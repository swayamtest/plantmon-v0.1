## Runtime Identity Governance Architecture

PLANTMON uses a layered runtime identity governance architecture to separate:

* onboarding recognition  
* operational normalization  
* canonical runtime identity  
* operational care intelligence

This separation is foundational to:

* onboarding usability  
* runtime scheduler safety  
* operational scalability  
* future AI extensibility

### **Identity Layer Hierarchy**

| Layer | Responsibility |
| ----- | ----- |
| canonical\_speciess | Immutable operational identity backbone |
| canonical\_inventory | Canonical operational species registry |
| collapse\_mappings | Operational normalization layer |
| plant\_aliases | Recognition and onboarding layer |
| plant\_care\_profiles | Runtime operational care intelligence |

---

### **Recognition vs Normalization Principle**

PLANTMON intentionally separates:

#### **plant\_aliases**

Aliases solve:  
recognition flexibility.

They allow users to search using:

* common names  
* regional names  
* nursery names  
* beginner terminology  
* consumer-facing naming patterns

Examples:

* Money Plant  
* Snake Plant  
* Tulsi  
* ZZ Plant

Aliases improve:

* onboarding speed  
* search usability  
* beginner confidence

Aliases DO NOT:  
control runtime operational scheduling.

### **Runtime Resolution Flow**

User Input  
→ Alias Recognition  
→ Collapse Normalization (optional)  
→ Canonical Species  
→ canonical\_species  
→ Runtime Care Intelligence

Aliases always resolve FIRST.

Operational normalization occurs ONLY after:  
plant recognition succeeds.

20. # INVENTORY LOCALIZATION PRINCIPLE

---

The inventory should strongly reflect:

* Indian households  
* balconies  
* terrace gardening  
* hot/humid climates  
* regional nursery economics  
* culturally important plants  
* utility plants  
* edible balcony gardening

The inventory should NOT over-index on:

* Western collector culture  
* rare aroids  
* greenhouse-only species

---

21. # OPERATIONAL DISTINCTIVENESS RULE

---

Distinct species should remain separate ONLY if they introduce:

* meaningful operational differences  
  OR  
* meaningful ownership recognition differences.

Operational normalization is permitted ONLY when:

* watering behavior remains safely aligned  
* humidity behavior remains safely aligned  
* repotting behavior remains safely aligned  
* seasonal operational logic remains safely aligned  
* scheduler drift risk remains acceptably low

Operationally distinct species MUST remain separate when:

* growth habit differs  
* watering methodology differs  
* environmental tolerance differs  
* foliar morphology meaningfully changes care behavior  
* lifecycle maintenance diverges

Examples of protected operational classes include:

* crawlers vs climbers  
* velvet foliage vs glossy foliage  
* fuzzy succulents vs smooth succulents  
* mounted epiphytes vs terrestrial plants
