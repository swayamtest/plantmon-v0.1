# PLANTMON — COLLAPSE GOVERNANCE RULESET v1

---

# **PURPOSE**

This document governs:  
`collapse_mappings.csv`

It defines:

* what MAY collapse  
* what MUST NEVER collapse  
* operational normalization boundaries  
* onboarding normalization philosophy  
* runtime safety protections

This document supersedes:  
taxonomy-first normalization logic.

---

# **1\. CORE GOVERNANCE PRINCIPLE**

Collapse mappings exist ONLY for:

* onboarding normalization  
* operational simplification  
* search resolution  
* beginner usability  
* deterministic runtime scheduling

Collapse mappings DO NOT exist for:

* taxonomic completeness  
* botanical lineage  
* genus preservation  
* scientific hierarchy simulation

---

## **1.1 Alias vs Collapse Separation Principle**

Aliases and collapse mappings solve fundamentally different problems.

Aliases exist for:

* onboarding recognition  
* search flexibility  
* regional terminology  
* nursery terminology  
* beginner usability

Collapse mappings exist for:

* operational normalization  
* care profile consolidation  
* runtime scheduling consistency

Aliases MUST NOT:  
perform operational normalization.

Collapse mappings MUST NOT:  
behave like recognition dictionaries.

Examples:

"Money Plant"  
→ alias resolution

"Philodendron scandens"  
→ operational collapse normalization

This separation is REQUIRED to preserve:

* onboarding clarity  
* runtime determinism  
* scheduler stability  
* canonical identity integrity

---

# **2\. PRIMARY NORMALIZATION LAW**

A collapse mapping is ONLY valid if:

A beginner user would reasonably expect these plants to behave similarly enough operationally that they can safely share the same runtime care identity.

Operational behavior ALWAYS overrides taxonomy.

---

# **3\. RUNTIME SAFETY PRIORITY**

Runtime operational safety is:  
NON-NEGOTIABLE.

If a normalization introduces:

* watering drift  
* humidity drift  
* repotting drift  
* airflow drift  
* structural maintenance drift  
* seasonal drift

the normalization MUST be rejected.

---

# **4\. PROTECTED OPERATIONAL CLASSES**

The following classes MUST NEVER be aggressively normalized.

These represent:  
runtime-sensitive behavioral divergences.

---

## **4.1 Structural Growth Habit Divergence**

NEVER collapse across:

* crawler  
* climber  
* shingler  
* trailing  
* upright  
* terrestrial spreader  
* mounted epiphyte

Examples:

| Unsafe Collapse | Reason |
| ----- | ----- |
| Philodendron gloriosum → Philodendron erubescens | crawler vs climber |
| Monstera dubia → Monstera deliciosa | shingling behavior divergence |
| trailing succulent → upright succulent | water retention \+ support divergence |

---

## **4.2 Foliar Morphology Divergence**

NEVER collapse across:

* velvet foliage  
* fuzzy foliage  
* trichome-heavy foliage  
* glossy wax foliage  
* glaucous foliage

Examples:

| Unsafe Collapse | Reason |
| ----- | ----- |
| Echeveria setosa → Echeveria elegans | fuzzy trichome moisture sensitivity |
| Anthurium crystallinum → glossy Anthurium | foliar humidity behavior differs |

---

## **4.3 Watering Method Divergence**

NEVER collapse across:

* top-water species  
* soak-and-drain species  
* mounted soak species  
* bottom-water species  
* mist-dominant species

Examples:

| Unsafe Collapse | Reason |
| ----- | ----- |
| Tillandsia → soil-based tropicals | rootless epiphyte |
| mounted orchid → terrestrial orchid | hydration methodology differs |

---

## **4.4 Environmental Tolerance Divergence**

NEVER collapse across:

* xerophytic drought-tolerant plants  
* moisture-retentive tropicals  
* humidity-sensitive foliage  
* monsoon-sensitive plants  
* heat-sensitive understory plants

Examples:

| Unsafe Collapse | Reason |
| ----- | ----- |
| moisture-loving fern → drought epiphyte | transpiration mismatch |
| velvet aroid → glossy canopy aroid | airflow \+ fungal sensitivity mismatch |

---

# **5\. HIGH-CONFIDENCE COLLAPSE CATEGORIES**

The following collapse categories are strongly encouraged.

---

## **5.1 Cosmetic Cultivar Normalization**

ALLOWED when:

* morphology is mostly cosmetic  
* runtime behavior is nearly identical  
* onboarding overlap is high

Examples:

| Safe Collapse Cluster |
| ----- |
| Aglaonema cultivars |
| common Syngonium variants |
| standard Snake Plants |

---

## **5.2 Nursery Umbrella Normalization**

ALLOWED when:

* nursery labeling already groups them together  
* consumers already treat them as one category  
* operational drift risk is low

Examples:

| Safe Collapse Cluster |
| ----- |
| Snake Plants |
| generic Echeverias |
| Chinese Evergreens |

---

## **5.3 Beginner Recognition Simplification**

ALLOWED when:

* scientific distinction adds no UX value  
* care logic remains stable  
* onboarding friction drops significantly

Examples:

| Safe Collapse Cluster |
| ----- |
| generic mint variants |
| common pothos variants |
| generic arrowhead vines |

---

# **6\. PRESTIGE / COLLECTOR EXCEPTION RULE**

Certain species MUST remain separate even if:  
operationally similar.

This preserves:

* onboarding trust  
* collector recognition  
* search intent specificity  
* emotional ownership fidelity

Examples include:

| Protected Collector Identity |
| ----- |
| Philodendron micans |
| Aglaonema pictum |
| Anthurium crystallinum |
| Monstera dubia |

These SHOULD retain:  
independent canonical identities.

---

# **7\. RECURSIVE COLLAPSE PROHIBITION**

Collapse mappings MUST terminate directly into:  
ONE canonical operational identity.

The system MUST NOT allow:

* multi-hop collapse chains  
* recursive normalization  
* chained operational inheritance

Examples of prohibited behavior:

Species A  
→ Species B  
→ Species C

All normalization must resolve directly into:  
the canonical operational identity target.

---

# **8\. SELF-REFERENTIAL COLLAPSE PROHIBITION**

ABSOLUTE RULE:

A species MUST NEVER:

* collapse into itself  
  OR:  
* collapse while simultaneously existing as an active canonical species

This creates:

* runtime ambiguity  
* scheduler conflict  
* onboarding instability  
* identity duplication

Self-referential mappings MUST be removed immediately.

---

# **9\. COLLAPSE TABLE DENSITY PHILOSOPHY**

The collapse table SHOULD remain:  
small,  
high-confidence,  
and intentional.

Preferred philosophy:

HIGH PRECISION  
over  
HIGH RECALL.

Over-normalization is significantly more dangerous than under-normalization.

---

# **10\. ONBOARDING GOVERNANCE PRINCIPLE**

Normalization SHOULD optimize for:

* beginner recognition  
* nursery realism  
* regional familiarity  
* onboarding confidence  
* search success rate

NOT:  
botanical sophistication.

---

# **11\. INDIA-SPECIFIC NORMALIZATION RULES**

Operational normalization MUST consider:

* Indian nursery inventory realities  
* balcony gardening prevalence  
* monsoon conditions  
* thermal load exposure  
* common household terminology

Examples:

HIGH-VALUE normalization:

* Snake Plant  
* Money Plant  
* Chinese Evergreen

LOW-VALUE normalization:

* obscure collector aroids  
* greenhouse-only cultivars  
* rare imported hybrids

---

# **12\. CANONICAL RUNTIME TERMINATION RULE**

ALL collapse mappings MUST terminate in:  
canonical\_species\_id

NOT:  
species\_name alone.

Operational normalization is only considered complete once:  
runtime identity resolves into:  
canonical operational identity.

This preserves:

* runtime stability  
* migration safety  
* scheduler consistency  
* long-term identity durability

# **13\. RUNTIME SCHEDULER SAFETY RULE**

The scheduler MUST NEVER inherit care logic from:  
behaviorally incompatible species.

If:  
watering,  
repotting,  
humidity,  
or seasonal behavior materially differs:

the species MUST retain:  
independent operational identity.
